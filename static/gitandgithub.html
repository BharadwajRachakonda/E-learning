Git and GitHub Commands

A complete guide to using Git and GitHub effectively

Windows Commands

  • $ cd {directory_name} - Change to a directory on the current drive
  • $ cd /d {d:\\programming\\html} - Change to a directory on another drive
  • $ dir - Display the contents of the current directory
  • $ mkdir - Create a new directory
  • $ dir /a:h - Display hidden files in a directory
  • $ echo. > file.txt - Create a new file
  • $ rm -rf {file_name} - Delete a file

Git Commands

  • $ git init - Initialize a new Git repository
  • $ git status - View changes in the repository
  • $ git add . - Stage all changes for commit
  • $ git commit -m "your text" - Commit changes with a message
  • $ git restore --staged {file_name} - Unstage changes
  • $ git log - View the commit history
  • $ git reset {sha_code} - Reset to a previous commit
  • $ git stash - Temporarily store changes
  • $ git remote add {origin} {URL} - Add a remote repository
  • $ git branch {branch_name} - Create a new branch
  • $ git checkout {branch_name} - Switch branches
  • $ git clone {URL} - Clone a remote repository

Merge Conflicts

A merge conflict occurs when multiple contributors modify the same line or content in a file. Resolve these conflicts manually before completing the merge.
